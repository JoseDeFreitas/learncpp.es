## Datos

En la [lección 01.01](/Contenido/Cap%C3%ADtulo%2001%20Fundamentos%20de%20C%2B%2B/01.01%20Declaraciones%20y%20la%20estructura%20de%20un%20programa.md)
aprendimos que la mayoría de las instrucciones de un programa son declaraciones y que éstas
a su vez son agrupadas dentro de funciones. Estas declaraciones realizan acciones que
generarán los resultados del programa.

Pero, ¿como hacen los programas para generar resultados? Estos generan resultados manipulando
(leyendo, cambiando y escribiendo) datos. En la informática, los **datos** son cualquier tipo
de información que puede ser movida, procesada o almacenada en y por una computadora.

> **Información clave**
>
> Los programas son colecciones de instrucciones que manipulan datos para generar un resultado
> deseado.

Los programas puede obtener datos de diversas maneras: desde un archivo de base de datos,
desde una network, desde el input del usuario a través del teclado o desde el input de un
programador que envía datos directamente dentro del código fuente. En el programa "hola mundo"
en la lección mencionada, el texto "¡Hola, mundo!" fue insertado directamente en el código
fuente del programa, haciendo que este pueda usar ese texto. El programa manipula este dato
al enviarlo al monitor para poder ser visto por el usuario.

Los datos son comunmente almacenados en la computadora en un formato eficiente para su
almacenamiento y procesamiento (por lo que no es muy legible por los humanos). Debido a esto,
cuando el programa "hola mundo" es compilador, el texto "¡Hola, mundo!" se convierte en un
formato más eficiente para que el programa pueda usarlo más fácilmente (sistema binario, el
cual discutiremos en una lección posterior).

## Objetos y variables

Todas las computadoras tienen memoria, conocida como **RAM** ("random access memory";
"memoria de acceso aleatorio" en español), la cual es usada por los programas. Puedes pensar
en la RAM como una serie de buzones de correo que son usados para almacenar datos mientras
el programa está en ejecución. Un dato único almacenado en alguna parte de la memoria es
llamado **valor**.

En lenguajes de programación antiguos (como Apple Basic) podrías acceder directamente
a estos buzones (una declaración podría decir "*ve y toma el valor almacenado en el buzón número 7532*").

En C++ no se puede acceder directamente a la memoria. En vez de eso, puede accederse a la
memoria indirectamente a través e un objeto. Un **objeto** es una parte del almacén (
usualmente memoria) que tiene un valor y otras propiedades relacionadas (hablaremos de esto
en lecciones futuras). Cuando se define un objeto el compilador determina automáticamente
en qué lugar de la memoria será almacenado. En vez de decir "*ve y toma el valor almacenado en el buzón número 7532*",
podemos decir: "*ve y toma el valor almacenado de este objeto*" y el compilador sabrá dónde
buscar el objeto en la memoria. Gracias a este funcionamiento podemos centrarnos en usar
objetos para almacenar y usar daots en vez de preocuparnos por el lugar exacto en memoria
donde el valor está almacenado.

Los objetos pueden tener nombre o no tener (objeto anónimos). Un objeto con nombre es llamado
**variable**, mientras que uno sin nombre es llamado **identificador**. La mayoría de objetos que
crearemos en nuestros programas serán variables.

> **Nota del autor**
>
> En términos de programación generales, el concepto *objeto* se refiere a una variable, una
> estructura de datos almacenada en memoria o una función. En C++ el concepto *objeto* tiene
> una definición menos amplia que excluye a las funciones.

## Inicialización de variables

Para crear una variable usamos un tipo de declaración especial llamado **definición** (
ahondaremos en la diferencia entre declaración y definición luego).

Este es un ejemplo de una definición de una variable llamada x:
```cpp
int x; // definición de una variable llamada x de tipo int
```

En el momento de compilación el compilador ve esta declaración y toma nota de que estamos
definiendo una variable llamada *x* y de tipo *int* (en un momento explicaremos los tipos).
A partir de ese punto (con algunas limitaciones que veremos en lecciones futuras) el
compilador sabrá que nos estamos refiriendo a la variable *x* cada vez que la encuentre a lo
largo de nuestro programa.

Cuando el programa es ejecutado (proceso llamado **tiempo de ejecución**) la variable será
inicializada. "**Inicialización**" es una palabra elegando que significa que se creará un
objeto y será asignado a una dirección en la memoria. Las variables deben ser inicializadas
antes de ser utilizadas para almacenar valores. Para entender esto, digamos que la variable
*x* está siendo inicializada en la dirección de memoria 140. Cada vez que el programa use la
variable *x* este tendrá acceso al valor en la dirección de memoria 140. Comunmente se le
llama **instancia** a los objetos inicializados.

## Tipos de datos

Hasta ahora hemos aprendido que las variables son objetos con nombre que pueden almacenar
datos (en futuras lecciones veremos cómo exactamente se almacenan los datos). Un
**tipo de dato** (también llamado simplemente "**tipo**") le dice a la computadora qué tipo
de valor almacenará la variable (por ejemplo: número, texto, etc.).

En el ejemplo de arriba la variable *x* es de tipo *int*, lo que quiere decir que la variable
*x* representará un valor entero. Un **entero** ("integer" en inglés) es un número que puede
ser escrito sin decimales, como 4, 27, 0, -2 o -12. Podemos decir que *x* es una *variable de entero*.

En C++, el tipo de una variable debe ser determinado en el **tiempo de compilación** (cuando
se compila un programa). El tipo de la variable no puede ser cambiado luego de compilar el
programa. Esto significa que una variable de tipo entero puede almacenar solo datos de tipo
entero. Si queires almacenar un valor de otro tipos tendrás que usar una variable diferente.

Los enteros son solo uno de los muchos tipos de datos que soporta C++. Para mostrarte un poco
más, aquí hay un ejemplo de la definición de una variable de tipo *double*:
```cpp
double witdh; // definición de una variable llamada width de tipo double
```

C++ también permite crear tus propios tipos de datos. Esto es algo que haremos mucho en
lecciones futuras ya que es algo que hace que C++ sea aun más poderoso.

En estos capítulos introductorios definiremos variables de tipo entero ya que son
conceptualmente simples, pero pronto exploraremos muchos otros tipos de datos en C++.

## Definiendo múltiples variables

Es posible definir más de una variable *del mismo tipo* en una sola línea de declaración
separando los nombres con comas. Las porciones de código siguientes funcionan igual:
```cpp
int a;
int b;
```
es lo mismo que:
```cpp
int a, b;
```

Cuando definimos más de una variable de esta manera podemos cometer dos errores comunes (
no errores graves ya que el compilador notará estos errores y hará que los corrijas).

El primer error está en darle un tipo de dato a cada variable al definirlas:
```cpp
int a, int b; // incorrecto (el compilador devolverá un error)

int a, b; // correcto
```

El segundo error está en intentar definir variables de tipos diferentes en una misma
declaración, lo cual no está permitido. Las variables de tipos de datos diferentes deben
ser definidas en declaraciones diferentes:
```cpp
int a, double b; // incorrecto (el compilador devolverá un error)

int a; double b; // correcto (pero no es recomendable)

// correcto y recomendable (ya que es más fácil de leer)
int a;
double b;
```

> **Mejores prácticas**
>
> A pesar de que el lenguaje te permite hacerlo, evita definir múltiples variables en una
> sola declaración (incluso si son del mismo tipo). Es preferible que definas cada variable
> en una declaración por separado (y luego uses un comentario de una sola línea para
> documentar para qué se usa esa variable).

## Resumen

En C++ usamos variables para acceder a la memoria de la computadora. Las variables tienen
un identificador, un tipo y un valor (y algún que otro atributo que no es relevante por el
momento). El tipo de una variable se usa para determinar cómo debe ser interpretado el valor
en la memoria.

En la siguiente lección veremos cómo asignarle valores a las variables y cómo usar las
variables.
