## Palabras reservadas

C++ tiene una lista de 92 palabras reservadas (a partir de C++20). Estas palabras se llaman
**palabras reservadas** y cada una de ellas tiene un significado dentro del lenguaje C++.

Aquí está la lista de todas las palabras reservadas de C++ (C++20):

Las palabras reservadas marcadas con "C++20" fueron añadidas en C++20. Si tu compilador no
soporta la versión C++20 (o si la soporta pero tiene la funcionalidad desactivada por defecto)
puede que no funcionen.

C++ también tiene algunos identificadores especiales: *override*, *final*, *import*, y
*module*. Estos tienen un significado cuando se usan en ciertos contextos.

Ya has visto alguna de estas palabras reservadas, como *int* y *return*. Además de un conjunto
de operadores, estas palabras reservadas y los identificadores especiales definen el lenguaje
C++ (excluyendo los comandos del preprocesador). Ya que estas palabras reservadas e
identificadores especiales tienen un significado, tu IDE cambiará el color de estas palabras
(generalmente a azul) para destacarlas del resto de palabras.

¡Entenderás lo que hace cada una de estas palabras al final de los tutoriales!

## Reglas del nombramiento de identificadores

Como recordatorio: el nombre de una variable (o de una función, tipo o cualquier otro
elemento) se llama identificador. C++ te da mucha libertar en cuanto a los nombres de los
identificadores. No obstante, al algunas reglas que debes seguir al nombrar objetos:
- El identificador no puede ser una palabra reservada.
- El identificador puede tener solo letras (minúscula o mayúscula), número y el caracter
guión bajo (_). Esto significa que el nombre no puede contener símbolos (exceptuando el
guión bajo) ni espacios en blanco.
- El identificador debe comenzar con una letra (minúscula o mayúscula) o un guión bajo. No
puede comenzar con un número.
- C++ distingue entre minúsculas y mayúsculas. `nvalue` es diferente a `nValue` el cual es a
su vez diferente a `NVALUE`.

## Mejores prácticas del nombramiento de identificadores

Ahora que sabes que *puedes* nombrar una variable, discutamos sobre cómo *deberías* nombrar
una variable (o función).

Primero, es una convención en C++ que los nombres de las variables empiecen con una letra
minúscula. Si la variable es una sola palabra entonces la palabra entera debe ser escrita en
minúscula.

```cpp
int value; // correcto
 
int Value; // incorrecto (debería empezar con minúscula)
int VALUE; // incorrecto (debería empezar con minúscula)
int VaLuE; // incorrecto (ve a un psiquiatra) ;)
```

En la mayoría de los casos los nombres de las funciones también empiezan con una letra
minúscula (aunque algunos no están de acuerdo en esto). Seguiremos esta convención ya que
*main* (la función que todos los programas deben tener) empieza con una letra minúscula, al
igual que todas las funciones de la librería estándar de C++.

Los nombres de los identificadores que empiezan con una letra mayúscula son comunmente usadas
para tipos definidos por el usuario (como estructuras, clases y enumeraciones, objetos que
explicaremos luego).

Si el nombre de la variable o función consta de más de una palabra hay dos convenciones
típicas: palabras separadas por guiones bajos, llamada snake_case, o palabras separadas por
mayúsculas (exceptuando la primera letra), llamada camelCase.

```cpp
int my_variable_name; // correcto (snake_case)
int my_function_name(); // correcto (snake_case)
 
int myVariableName; // correcto (camelCase)
int myFunctionName(); // correcto (camelCase)
 
int my variable name; // inválido (no se permiten los espacios en blanco)
int my function name(); // inválido (no se permiten los espacios en blanco) 
 
int MyVariableName; // válido pero incorrecto (debería empezar con minúscula)
int MyFunctionName(); // válido pero incorrecto (debería empezar con minúscula)
```

En este tutorial usaremos camelCase porque es más fácil de leer (es fácil confundir un
espacio con un guión bajo en códigos densos). Aún así, es común ver ambas convenciones. La
librería estándar de C++ usa el guión bajo para variables y funciones. A veces verás una
mezcla de ambas: snake_case para variables y camelCase para funciones.

Es mejor que si estás trabajando en el código de alguien más te adaptes al estilo del código
en vez de seguir estrictamente las convenciones de arriba.

> **Mejores prácticas**
>
> Cuando estés trabajando en un programa existente usa las convenciones de ese programa
> (a pesar de que no estés conforme con las mejores prácticas). Usa las prácticas modernas
> cuando estés escribiendo tus programas.

Segundo, debes evitar nombrar tus identificadores empezando con un guión bajo porque este
tipo de nombres estás reservados por el sistema operativo o el compilador.

Tercero, el nombre de los identificadores debe reflejar qué valor contienen (especialmente
si el valor no es obvio). Los identificadores deben tener un nombre que le diga a una persona
que no entiende tu código que esa variable es cierta cosa. En 3 meses cuando leas tu programa
de nuevo te darás cuenta de que te has olvidado de cómo funciona y estarás agradecido contigo
mismo por haber nombrado a tus variables de manera lógica.

Sin embargo, si le das un nombre complejo a una variable trivial haces que tu código sea más
difícil de entender tanto como si le das un nombre inadecuado a un identificador ampliamente
usado. Por lo tanto, una buena regla general es que hagas que la longitud del nombre del
identificador sea proporcional a qué tanto es usado. Un identificador de uso trivial puede
tener un nombre corto (como *i*). Un identificador que es usado generalmente (por ejemplo:
una función que es llamada en muchos lugares de tu programa) puede tener un nombre más
largo y más descriptivo (ej: en vez de *open*, *openFileOnDisk*).

| Nombre             | Práctica | Descripción                                       |
| ------------------ | -------- | ------------------------------------------------- |
| int ccount         | Mal      | ¿Qué significa la "c" antes de "count"?           |
| int customerCount  | Bien     | Es claro lo que estamos contando                  |
| int i              | Depende  | Está bien si es trivial; si no, está mal          |
| int index          | Depende  | Está bien si es obvio lo que estamos indexando    |
| int totalScore     | Depende  | Está bien si solo calificamos una cosa, si no, no |
| int _count         | Mal      | No empieces nombres con un guión bajo             |
| int count          | Depende  | Está bien si es obvio lo que estamos contando     |
| int data           | Mal      | ¿Qué tipo de "data" (datos)?                      |
| int time           | Mal      | ¿Segundos, minutos u horas?                       |
| int minutesElapsed | Bien     | Es descriptivo                                    |
| int value1, value2 | Depende  | Puede ser difícil diferenciarlos                  |
| int numApples      | Bien     | Es descriptivo                                    |
| int monstersKilled | Bien     | Es descriptivo                                    |
| int x, y           | Depende  | Está bien si son triviales; si no, está mal.      |

**Como se expone en el capítulo 01.01, al programar se suele escribir todo en inglés salvo el texto que verá el usuario o los comentarios si nuestro programa va dirigido a personas de habla hispana; es por eso que las variables están en inglés. Aún así, puedes escribirlas en español, aunque te recomiendo que uses inglés salvo para lo anteriormente mencionado.**

En cualquier vaso, evita las abreviaciones. Si bien reducen el tiempo que necesitas para
escribir el código, pueden hacer que tu código sea difícil de leer. Incluso si la abreviación
no es ambigua el lector debe prestar más atención para determinar a qué se refiere el nombre.
El código se lee más que se escribe, por lo que el tiempo que salvaste al escribir nombres
abreviados está siendo gastado en el futuro por los que lean tu código y por ti mismo. Si
quieres escribir código más rápido usa la opción de completación automática de tu IDE.

Para terminar: puedes usar comentarios para ser más claro. Por ejemplo, hemos declarado una
variables llamada *numberOfChars*, que se supone que almacene el número de caracteres de
un texto. ¿Cuántos caracteres tiene el texto "¡Hola, mundo!", 9, 10, 12 o 13? Depende de si
estás incluyendo los espacios en blanco o los signos ortográficos. En vez de llamar a la
variable *numberOfCharsIncludingWhitespaceAndPunctuation*, el cual es bastante largo,
podemos escribir un comentario sobre la declaración de la variable para darle más información
al usuario:
```cpp
// almacena el número de caracteres en un texto (¡incluyendo los espacios en blanco y los signos ortográticos!)
int numberOfChars;
```
