El **espacio en blanco** es un término que se refiere a los caracteres que se usan para
formatear un documento. En C++ tenemos principalmetne los espacios, el tabulador y las nuevas
líneas. El compilador de C++ generalmente ignora el espacio en blanco, exceptuando algunas
cosas (cuando se procesan strings de texto literales). Por esta razón es que decimos que C++
es un lenguaje que no depende del espacio en blanco.

> **Nota**
>
> - Con "formatear" me refiero a "dar un formato o presentación a una tabla numérica o a un
> documento". Hago esta aclaratoria porque este término se ve aplicado con el significado de
> "eliminar los datos de un disco", por ejemplo. Me baso en la [definición de la RAE](https://dle.rae.es/formatear).
> - Los espacios en blanco comprenden: espacios (barra espaciadora), tabuladores (tecla TAB,
> al lado izquierdo de la tecla "Q" en un teclado español) y nuevas líneas (al presionar la
> tecla "Enter" cuando estás en un campo de texto).

Las siguientes declaraciones hacen lo mismo:
```cpp
std::cout << "¡Hola, mundo!";
 
std::cout               <<            "¡Hola, mundo!";
 
		std::cout << 		"¡Hola, mundo!";
 
std::cout
	<< "¡Hola, mundo!";
```

Incluso la última declaración que comprende dos líneas funciona perfectamente.

Las siguientes funciones hacen lo mismo:
```cpp
int add(int x, int y) { return x + y; }
 
int add(int x, int y) {
    return x + y; }
 
int add(int x, int y)
{    return x + y; }
 
int add(int x, int y)
{
    return x + y;
}
```

Una excepción de ignorar el espacio en blanco que tiene el compilador de C++ es dentro de
un string (""), como `"¡Hola, mundo!"`.

`"¡Hola, mundo!"` es diferente a `"¡Hola,     mundo!"`, y al imprimirlos ambos devuelven el
resultado que esperas.

Las nuevas líneas no se permisen en un string:
```cpp
std::cout << "¡Hola,
     mundo!"; // ¡no es correcto!
```

Los strings separados por espacios en blanco (espacios, tabuladores o nuevas líneas) serán
concatenados:
```cpp
std::cout << "¡Hola, "
     "mundo!"; // imprime "¡Hola, mundo!"
```

Otra excepción es que el compilador de C++ toma en cuenta los espacios en blanco en los
comentarios de una línea. Los comentarios de una línea terminan al final de la línea. Es por
esto que si haces algo como esto obtendrás algunos errores:
```cpp
std::cout << "¡Hola, mundo!"; // este es un comentario de una línea
esto no es parte del comentario
```

## Formateo básico

A diferencia de otros lenguajes de programación, C++ no te obliga a seguir reglas de formateo
al programar (recuerda: ¡confía en el programador!). Se han desarrollado muchos métodos de
formateo en C++, y estarás en desacuerdo con muchos de ellos. La regla general que
recomendamos es que los mejores estilos son los que producen un código más legible y proveen
mayor consistencia.

Estas son nuestras recomendaciones del formateo básico:
1. Está bien usar tabuladores y espacios para aplicar sangría (la mayoría de los IDE vienen
con una opción que transforma automáticamente los tabuladores al número correcto de espacios).
Los desarrolladores que prefieren usar espacios tienden a usarlos porque hace que el formato
se describa a sí mismo; el código que es formateado usando espacios siempre se verá bien
independientemente del editor de texto. Los que prefieren usar tabuladores se preguntan:
"¿Por qué no usarías el caracter diseñado para aplicar sangría, especialmente cuando puedes
configurar la longitud?". No hay una respuesta correcta a esto; este debate es como discutir
si un pastel el más delicioso que un pie o viceversa. Al fin y al cabo es de preferencia
personal.
Te recomendamos que configures tus tabuladores a 4 espacios de sangría. Algunos IDE configuran
los tabuladores a 3 espacios por defecto. 3 espacios también está bien.
2. Hay dos estilos aceptables para escribir las llaves.
La guía de estilos de Google C++ recomienda que coloques las lalves en la misma línea que
la declaración:
```cpp
int main() {
}
```
Justifican esta práctica porque reduce la cantidad de espacios en blanco (no estás destinando
una línea entera para un solo caracter), por lo que puedes ahorrarte unas líneas de código.
Mientras más código haya en la pantalla más fácil es de entender.
Sin embargo, nosotros preferimos una alternativa, en donde cada llave tiene su línea:
```cpp
int main()
{
}
```
Esto hace que el código sea más legible y evita algunos errores ya que las llaves deben te
tener la misma sangría al abrir y cerrar la función. Si recibes un error de compilación
debido a que algunas llaves no cierran, es más fácil darse cuenta dónde está ocurriendo el
error.
3. Cada declaración dentro de unas llaves debe tener una sangría de un tabulador en relación
a las llaves. Por ejemplo:
```cpp
int main()
{
    std::cout << "¡Hola, mundo!\n"; // con sangría de un tabulador (4 espacios)
    std::cout << "Encantado de conocerte.\n"; // con sangría de un tabulador (4 espacios)
}
```
4. Las líneas no deben ser muy largas. Comunmente la máxima longitud de una línea de código
es de 80 caracteres. Si la línea es más larga se partirá (en un lugar lógico) en más líneas.
Puedes hacer esto aplicando sangría a cada línea consecuente con un tabulador extra, o, si las
líneas son similares, puedes alinearlas con la línea de arriba (la que sea más fácil de leer):
```cpp
int main()
{
    std::cout << "Esta es una línea muy, muy, muy, muy, muy, muy, muy, muy, muy, muy, muy, " 
        "muy larga\n"; // un tabulador extra para continuar la línea
 
    std::cout << "Esta es una línea muy, muy, muy, muy, muy, muy, muy, muy, muy, muy, muy, "
                 "muy larga\n"; // texto alineado a la línea anterior para continuar la línea
 
    std::cout << "Esta es corta\n";
}
```
Esto hace que las líneas sean más fácil de leer. En los monitores modernos también permite
poner dos ventanas con código similar una al lado de otra para comparar el código de manera
más sencilla.
> **Mejores prácticas**
>
> Las líneas no deben ser de más de 80 caracteres de longitud.
5. Si unal ínea muy larga de parte con un operador (por ejemplo, << o +), éste debe ser
colocado al comienzo de la línea, no al final de la línea principal:
```cpp
    std::cout << 3 + 4
        + 5 + 6
        * 7 * 8;
```
De esta manera se puede entender con más facilidad que las líneas consecuentes son
continuaciones de las líneas previas y te permite alinear los operadores a la izquierda para
una mejor legibilidad.
6. Usa espacios en blanco para hacer que tu código se lea más fácil al alinear los valores o
comentarios o al añadir especios entre porciones de código.
Difícil de leer:
```cpp
cost = 57;
pricePerItem = 24;
value = 5;
numberOfItems = 17;
```
Fácil de leer:
```cpp
cost          = 57;
pricePerItem  = 24;
value         = 5;
numberOfItems = 17;
```
Difícil de leer:
```cpp
std::cout << "¡Hola, mundo!\n"; // cout reside en la librería iostream
std::cout << "¡Es un placer conocerte!\n"; // estos comentarios son difíciles de leer,
std::cout << "¡Yeah!\n"; // especialmente cuando las líneas tienen longitudes diferentes
```
Fácil de leer:
```cpp
std::cout << "¡Hola, mundo!\n";             // cout reside en la librería iostream
std::cout << "¡Es un placer conocerte!\n";  // estos comentarios son fáciles de leer,
std::cout << "¡Yeah!\n";                    // especialmente cuando están alineados
```
Difícil de leer:
```cpp
// cout reside en la librería iostream
std::cout << "¡Hola, mundo!\n";
// estos comentarios son difíciles de leer,
std::cout << "¡Es un placer conocerte!\n";
// especialmente cuando están todos pegados
std::cout << "Yeah!\n";
```
Fácil de leer:
```cpp
// cout reside en la librería iostream
std::cout << "¡Hola, mundo!\n";

// estos comentarios son fáciles de leer,
std::cout << "¡Es un placer conocerte!\n";

// especialmente cuando están separados por espacios en blanco
std::cout << "Yeah!\n";
```
Seguiremos estas convenciones a lo largo de los tutoriales, haciendo que se te graben en la
memoria. Cuando te vayamos enseñando nuevos temas te enseñaremos también nuevas
recomendaciones de estilos que complementan a esos temas.

Para terminar: C++ te permite elegir el estilo que más te guste o el que creas que es el
mejor. No obstante, te recomendamos usar el mismo estilo que usamos en nuestros ejemplos. Este
estilo ha sido probado por miles de programadores en millardos de líneas de código y ha sido
optimizado para un buen rendimiento. Solo hay una excepción: si estás trabajando con el código
de alguien más, sigue los estilos que tiene. La consistencia es mejor que tus gustos.

## Formateo automático

La mayoría de los IDE modernos te ayudarán a formatear tu código mientras vas escribiendo
(ej: cuando creas una función el IDE le aplicará la sangría automáticamente a las
declaraciones dentro de la función).

Sin embargo, a medida que vas añadiendo y removiendo código, o si cambias la configuración
inicial de tu IDE, o si pegas una porción de código que tiene un formato diferente, el formato
de tu código puede fallar. Puede llegar a ser un verdadero dolor de cabeza el intentar
solucionar el formateo de tu código. Afortunadamente, la mayoría de los IDE tendrán
funciones de formateo que solucionarán porciones de código con un formato erróneo o el archivo
entero.

> **Para usuarios de Visual Studio**
>
> En Visual Studio puedes encontrar las opciones del formateo automático en *Edit > Advanced > Format Document and Edit > Advanced > Format Selection*.

> **Para usuarios de Code::Blocks**
>
> En Code::Blocks puedes encontrar las opciones del formateo automático haciendo *clic derecho con el mouse > Format use AStyle*.

Para acceder más rápido a estas opciones te recomendamos añadir un atajo del teclado para
formatear automáticamente una porción de código o un archivo entero.

> **Mejores prácticas**
>
> Es muy recomendable usar la opción del formateo automático para mantener el formato de tu
> código acorde al estilo que estás siguiendo.
